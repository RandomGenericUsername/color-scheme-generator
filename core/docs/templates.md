## Template System Overview

Colorscheme-generator uses **Jinja2** for template rendering. Templates define how color schemes are formatted and written to files.

### How It Works

```
┌──────────────┐
│ ColorScheme  │  (Python object with colors)
└──────┬───────┘
       │
       ↓
┌──────────────┐
│   Template   │  (Jinja2 .j2 file)
│  colors.json │
└──────┬───────┘
       │
       ↓ (Jinja2 rendering)
       │
┌──────────────┐
│ Output File  │  (Generated file)
│  colors.json │
└──────────────┘
```

### Template Location

**Built-in templates**:
```
core/src/colorscheme_generator/templates/
├── colors.json.j2
├── colors.sh.j2
├── colors.css.j2
├── colors.gtk.css.j2
├── colors.yaml.j2
├── colors.toml.j2
├── colors.sequences.j2
└── colors.rasi.j2
```

**Custom templates**:
```
~/.config/colorscheme-generator/templates/
├── colors.json.j2       # Override built-in
├── colors.css.j2        # Override built-in
└── my-custom-format.j2  # New format
```

## Built-in Templates

### JSON Template (`colors.json.j2`)

```json
{
  "special": {
    "background": "{{ background.hex }}",
    "foreground": "{{ foreground.hex }}",
    "cursor": "{{ cursor.hex }}"
  },
  "colors": {
    {% for color in colors -%}
    "color{{ loop.index0 }}": "{{ color.hex }}"{{ "," if not loop.last else "" }}
    {% endfor %}
  },
  "metadata": {
    "source_image": "{{ metadata.source_image }}",
    "backend": "{{ metadata.backend }}",
    "generated_at": "{{ metadata.generated_at }}"
  }
}
```

### Shell Template (`colors.sh.j2`)

```bash
#!/bin/sh
# Generated by colorscheme-generator
# Source: {{ metadata.source_image }}
# Backend: {{ metadata.backend }}

# Special colors
background='{{ background.hex }}'
foreground='{{ foreground.hex }}'
cursor='{{ cursor.hex }}'

# ANSI colors
{% for color in colors -%}
color{{ loop.index0 }}='{{ color.hex }}'
{% endfor %}
```

### CSS Template (`colors.css.j2`)

```css
/* Generated by colorscheme-generator */
/* Source: {{ metadata.source_image }} */
/* Backend: {{ metadata.backend }} */

:root {
  /* Special colors */
  --background: {{ background.hex }};
  --foreground: {{ foreground.hex }};
  --cursor: {{ cursor.hex }};

  /* ANSI colors */
  {% for color in colors -%}
  --color{{ loop.index0 }}: {{ color.hex }};
  {% endfor %}
}
```

## Template Variables

### Available Variables

All templates have access to these variables:

| Variable | Type | Description | Example |
|----------|------|-------------|---------|
| `background` | Color | Background color | `{{ background.hex }}` |
| `foreground` | Color | Foreground color | `{{ foreground.hex }}` |
| `cursor` | Color | Cursor color | `{{ cursor.hex }}` |
| `colors` | List[Color] | 16 ANSI colors | `{{ colors[0].hex }}` |
| `metadata` | Metadata | Generation metadata | `{{ metadata.backend }}` |

### Color Object Properties

Each `Color` object has:

| Property | Type | Description | Example |
|----------|------|-------------|---------|
| `hex` | str | Hex color code | `#1a1b26` |
| `rgb` | tuple | RGB values (0-255) | `(26, 27, 38)` |
| `hsl` | tuple | HSL values (optional) | `(235, 0.19, 0.13)` |

**Usage**:
```jinja2
{{ background.hex }}           {# #1a1b26 #}
{{ background.rgb[0] }}        {# 26 #}
{{ background.rgb[1] }}        {# 27 #}
{{ background.rgb[2] }}        {# 38 #}
```

### Metadata Object Properties

| Property | Type | Description | Example |
|----------|------|-------------|---------|
| `source_image` | str | Source image path | `/home/user/wallpaper.png` |
| `backend` | str | Backend used | `pywal` |
| `generated_at` | str | ISO timestamp | `2025-12-11T10:30:00` |

**Usage**:
```jinja2
{{ metadata.source_image }}    {# /home/user/wallpaper.png #}
{{ metadata.backend }}         {# pywal #}
{{ metadata.generated_at }}    {# 2025-12-11T10:30:00 #}
```

## Creating Custom Templates

### Step 1: Create Template Directory

```bash
mkdir -p ~/.config/colorscheme-generator/templates
```

### Step 2: Create Template File

Template filename format: `colors.<format>.j2`

Example: `colors.i3.j2` for i3 window manager config

### Step 3: Write Template

```jinja2
# ~/.config/colorscheme-generator/templates/colors.i3.j2

# i3 config - Generated by colorscheme-generator
# Source: {{ metadata.source_image }}

# class                 border      background  text        indicator   child_border
client.focused          {{ colors[4].hex }} {{ colors[4].hex }} {{ foreground.hex }} {{ cursor.hex }}   {{ colors[4].hex }}
client.focused_inactive {{ colors[0].hex }} {{ colors[0].hex }} {{ foreground.hex }} {{ colors[0].hex }} {{ colors[0].hex }}
client.unfocused        {{ colors[0].hex }} {{ colors[0].hex }} {{ colors[7].hex }} {{ colors[0].hex }} {{ colors[0].hex }}
client.urgent           {{ colors[1].hex }} {{ colors[1].hex }} {{ foreground.hex }} {{ colors[1].hex }} {{ colors[1].hex }}
client.placeholder      {{ colors[0].hex }} {{ colors[0].hex }} {{ foreground.hex }} {{ colors[0].hex }} {{ colors[0].hex }}

client.background       {{ background.hex }}
```

### Step 4: Configure

```toml
# settings.toml
[templates]
directory = "$HOME/.config/colorscheme-generator/templates"
```

### Step 5: Generate

```bash
colorscheme-gen generate wallpaper.png --formats i3
```

## Testing Templates

### 1. Generate with Verbose Output

```bash
colorscheme-gen generate test.png --verbose --formats myformat
```

### 2. Check Output File

```bash
cat ~/.cache/colorscheme/colors.myformat
```

### 3. Validate Syntax

For YAML/TOML/JSON templates:

```bash
# YAML
yamllint ~/.cache/colorscheme/colors.yaml

# TOML
python -c "import tomli; tomli.load(open('colors.toml', 'rb'))"

# JSON
jq . ~/.cache/colorscheme/colors.json
```

### 4. Test with Different Images

```bash
# Test with various images
for img in test1.png test2.jpg test3.png; do
  colorscheme-gen generate "$img" --formats myformat
  cat ~/.cache/colorscheme/colors.myformat
done
```

## Troubleshooting

### Template Not Found

**Problem**: `TemplateNotFound: colors.myformat.j2`

**Solutions**:
1. Check filename: Must be `colors.<format>.j2`
2. Check directory:
   ```bash
   ls ~/.config/colorscheme-generator/templates/
   ```
3. Verify config:
   ```toml
   [templates]
   directory = "$HOME/.config/colorscheme-generator/templates"
   ```

### Undefined Variable Error

**Problem**: `UndefinedError: 'undefined_var' is undefined`

**Solutions**:
1. Check variable name spelling
2. Verify variable exists (see [Template Variables](#template-variables))


### Syntax Error

**Problem**: `TemplateSyntaxError: unexpected '}'`

**Solutions**:
1. Check Jinja2 syntax
2. Ensure balanced `{% %}`, `{{ }}`, `{# #}`
3. Test template syntax:
   ```python
   from jinja2 import Environment
   env = Environment()
   env.from_string(open('template.j2').read())
   ```

## Best Practices

### 1. Add Comments

```jinja2
{# This template generates i3 window manager config #}
{# Color4 is used for focused window borders #}
client.focused {{ colors[4].hex }} ...
```

### 2. Use Semantic Names

```jinja2
{# Define semantic colors #}
{% set primary = colors[4] %}
{% set secondary = colors[6] %}
{% set danger = colors[1] %}
{% set success = colors[2] %}

# Use semantic names
border_color: {{ primary.hex }}
error_color: {{ danger.hex }}
```

### 3. Include Metadata

```jinja2
# Generated by colorscheme-generator
# Source: {{ metadata.source_image }}
# Backend: {{ metadata.backend }}
# Date: {{ metadata.generated_at }}
```

### 4. Validate Output

For structured formats (JSON, YAML, TOML), ensure valid syntax:

```jinja2
{# JSON - ensure proper commas #}
{
  "colors": {
    {% for color in colors -%}
    "color{{ loop.index0 }}": "{{ color.hex }}"{{ "," if not loop.last else "" }}
    {% endfor %}
  }
}
```
