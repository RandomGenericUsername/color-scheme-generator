## 13. Summary of Discrepancies

### Critical (Wrong Information)

1. **Auto-detection order**: 5 documentation files (`settings-schema.md`, `defaults.md`, `enums.md`, `core-commands.md`, `settings-layers.md`) state `pywal > wallust > custom`. Tests and `architecture.md`/`factory.md` confirm the correct order is **wallust > pywal > custom**.

2. **Default pywal algorithm**: `defaults.md` states `"haishoku"`. Both `backends.md` and `test_defaults.py` confirm the actual default is **`"wal"`**.

3. **Pywal algorithm list**: `backends-explained.md` lists `haishoku, colorz, colordb, eart` as valid values. The actual validated list (from `test_config.py`) is **`wal, colorz, colorthief, haishoku, schemer2`**. (`colordb`, `eart` are invalid; `wal`, `colorthief`, `schemer2` are missing from that doc.)

4. **Wallust backend_type validation**: Three documentation files list different sets of valid values (`none/adaptive/resized`, `resized/full`, or `resized/full/wal`). Test `test_backend_type_no_validation` confirms there is **no enum validation** ‚Äî any string is accepted.

5. **Saturation lower bound**: `settings-schema.md` implies `ge=0.5`. `test_config.py` parametrize and `types.md` confirm the real lower bound is **0.0**.

6. **Default backend in `settings-api.md`**: Example code shows `default_backend = "custom"`. Every other doc and `test_defaults.py`/`test_settings.py` confirm the correct default is **`"pywal"`**.

7. **Environment variable format**: `settings-layers.md` uses `COLOR_SCHEME_*` format. The actual format, confirmed by `run-smoke-tests.sh` and `use-dry-run.md`, is **`COLORSCHEME_<NAMESPACE>__<KEY>`** (double underscore).

### Significant Omissions

8. **`--dry-run` missing from `orchestrator-commands.md`**: The generate and show options tables in `orchestrator-commands.md` omit `--dry-run`, even though tests confirm it works.

9. **`SchemaRegistry.register()` argument count**: `packages/settings/README.md` shows 2 arguments; `settings-api.md` and tests show 3 (`namespace`, `model`, `defaults_file`).

10. **Non-existent CLI flags**: `container-setup.md` and `backends-explained.md` document `--seed`, `--algorithm`, `--clusters`, `--template-dir`, `--output-file` flags for `color-scheme-core`. None appear in `core-commands.md` or any test.

11. **CI workflow filenames**: `DEVELOPMENT.md` and `tests/smoke/README.md` reference `smoke-test-custom.yml`, `smoke-test-pywal.yml`, `smoke-test-wallust.yml`. The repository only contains `smoke-tests.yml`.

### Minor / Structural Issues

12. **6 broken internal doc links** (listed in Section 11).
13. **2 package READMEs** describe "Phase 1 - Under Development" state that no longer reflects reality.
14. **`developer-setup.md`** missing `color-scheme-templates` package from repository structure.
15. **Layer count inconsistency**: Some docs say 4 layers, `packages/settings/README.md` says 5 (adds environment layer). The 5-layer description is more accurate given env var support confirmed in tests.
16. **`ColorAlgorithm.DOMINANT`** is a valid config value (confirmed by tests) but is documented as NOT IMPLEMENTED ‚Äî that warning is correct but the existence as a valid config input is not clearly explained.
